Run a usage insights analysis and preserve the findings.

## Steps

1. Execute the `/insights` command to generate a usage analysis report
2. Review the generated HTML report for:
   - **What's Working**: Effective workflows and patterns
   - **Friction Points**: Common obstacles and slowdowns
   - **Quick Wins**: Easy improvements to implement now
   - **Ambitious Workflows**: Advanced patterns for future improvement
3. Create a summary in `docs/audit/insights-{{date}}.md` with:
   - Key metrics (sessions, tool usage, model distribution)
   - Top 3 friction points with proposed solutions
   - Top 3 effective patterns to reinforce
   - Action items for workflow optimization
4. If this is a monthly review, compare with previous insights reports
